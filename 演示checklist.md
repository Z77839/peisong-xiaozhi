## 🎯 核心话术

### 开场
> 「配送小智」是基于多 Agent 协同的 AI 配送运营智能体。从被动响应升级为主动预防。
> 4 城市 / 5 运力线 / 8 Agent 协同 / 3 个真实数学优化引擎 / 4 个真实数据接入端口。
> **已经接入和风天气真实 API**，其他 3 个等公司授权（Adapter 模式，业务代码 0 改动）。

### 决策中心
> 这是 8 Agent 协同决策。点"决策"按钮，看 8 个 Agent 逐步激活：任务路由 → 订单预测 → 运力分析 → 成本分析 → 派单推荐 → C 端增长 → 决策汇总 → 报告生成。
> 注意底部"Agent 调用追踪"，这是借鉴 Langfuse 设计的可观测性 — 没有这个，Agent 系统没法排查问题。

### 数据接入
> 这里展示 4 个真实数据接入端口的状态。**和风天气已经接了真实 API**。看我切城市，4 个城市的天气都是**实时拉的**。
> 其他 3 个饿了么/骑手/订单流是 Mock，但**架构已经就绪**。等公司授权就能切真实 API，**业务代码 0 改动**。
> 这就是 Adapter 模式的价值 — 数据源可换，业务不变。

### 生产就绪度
> 这是**生产就绪度仪表盘**，5 维度评分。综合 66/100，**准生产**。
> 投入公司前还需要 4 件事：3 个数据接入 + 公司云资源 + 监控告警 + 培训。
> 我们已准备好 5 份对接方案，就等公司授权。
