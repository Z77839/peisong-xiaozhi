# 配送小智 - 生产部署完整指南

> **目标**：在公司生产环境部署完整可用版本
> **预计时间**：2-4 小时

## 📋 部署前置条件

### 硬件要求（推荐）
- CPU: 2 核
- 内存: 4GB+
- 磁盘: 20GB+（含数据库）
- 操作系统: Ubuntu 22.04 / CentOS 8+

### 软件要求
- Docker 20.10+
- Docker Compose 2.0+
- 端口: 3000（应用）+ 5432（数据库）

### 域名 / HTTPS
- 一个域名（如 `peisong.example.com`）
- SSL 证书（Let's Encrypt 免费）

## 🚀 5 步部署

### 步骤 1：准备服务器

```bash
# 1. SSH 登录服务器
ssh root@your-server-ip

# 2. 安装 Docker
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker

# 3. 安装 Docker Compose
apt install docker-compose-plugin   # Ubuntu
# 或
yum install docker-compose-plugin   # CentOS

# 4. 验证
docker --version
docker compose version
```

### 步骤 2：上传代码

```bash
# 方式 A: 用 SCP 上传 peisong-local.tar.gz
scp peisong-local.tar.gz root@your-server-ip:/opt/

# 方式 B: 用 Git
ssh root@your-server-ip
cd /opt
git clone https://github.com/your-org/peisong-xiaozhi.git

# 登录服务器解压
cd /opt
tar -xzf peisong-local.tar.gz
mv peisong-xiaozhi peisong
cd peisong
```

### 步骤 3：配置环境变量

```bash
# 创建 .env 文件
cat > .env << 'EOF'
# 必须设置
JWT_SECRET=<运行 node -e "console.log(require('crypto').randomBytes(64).toString('hex'))" 得到的字符串>
DB_PASSWORD=<强密码，至少 32 字符>
CORS_ORIGIN=https://peisong.example.com

# 可选（暂时未接入可以留空）
QWEATHER_API_KEY=
QWEATHER_API_HOST=
ELEME_APP_KEY=
RIDER_WS_URL=
ORDER_STREAM_URL=
EOF

# 设置文件权限
chmod 600 .env
```

### 步骤 4：启动服务

```bash
# 构建 + 启动
docker compose up -d --build

# 查看启动日志
docker compose logs -f app

# 等 30 秒后查看状态
docker compose ps
```

**期望看到**：
```
NAME           STATUS          PORTS
peisong-app    Up (healthy)    0.0.0.0:3000->3000/tcp
peisong-db     Up (healthy)    5432/tcp
```

### 步骤 5：初始化数据库

首次启动会自动执行 `backend/migrations/001_init.sql`，但密码 hash 是 placeholder。需要手动初始化密码：

```bash
# 进入 app 容器
docker compose exec app sh

# 用 node 生成 bcrypt hash
node -e "
const bcrypt = require('bcryptjs');
const hash = bcrypt.hashSync('admin@2024', 12);
console.log(hash);
"

# 复制输出的 hash，然后执行 SQL
docker compose exec db psql -U peisong -d peisong -c "
UPDATE users SET password_hash = '\$2a\$12\$你的hash' WHERE username = 'admin';
UPDATE users SET password_hash = '\$2a\$12\$你的hash' WHERE username = 'operator';
UPDATE users SET password_hash = '\$2a\$12\$你的hash' WHERE username = 'analyst';
"
```

或者用更简单的方式 — **直接通过 API 重置**（需要 JWT）：

```bash
# 1. 启动后查看默认密码
curl http://localhost:3000/api/auth/demo-users
# 返回 admin@2024 / operator@2024 / analyst@2024

# 2. 登录
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"account":"admin","password":"admin@2024"}'
# 返回 token

# 3. 修改密码
curl -X POST http://localhost:3000/api/auth/change-password \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的token>" \
  -d '{"oldPassword":"admin@2024","newPassword":"你的新密码"}'
```

## 🔒 配置 HTTPS（Nginx 反向代理）

### 步骤 1：安装 Nginx + Certbot

```bash
apt install nginx certbot python3-certbot-nginx
systemctl start nginx
```

### 步骤 2：申请证书

```bash
certbot --nginx -d peisong.example.com
```

### 步骤 3：配置 Nginx

`/etc/nginx/sites-available/peisong`：

```nginx
upstream peisong_app {
  server localhost:3000;
}

server {
  listen 80;
  server_name peisong.example.com;
  return 301 https://$server_name$request_uri;
}

server {
  listen 443 ssl http2;
  server_name peisong.example.com;

  ssl_certificate /etc/letsencrypt/live/peisong.example.com/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/peisong.example.com/privkey.pem;

  # 安全头
  add_header X-Frame-Options "SAMEORIGIN" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-XSS-Protection "1; mode=block" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

  # 客户端最大上传
  client_max_body_size 10M;

  # 代理
  location / {
    proxy_pass http://peisong_app;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_cache_bypass $http_upgrade;
  }
}
```

```bash
ln -s /etc/nginx/sites-available/peisong /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

## 📊 配置监控告警

### 1. Prometheus + Grafana（可选）

`docker-compose.monitor.yml`：

```yaml
version: '3.8'
services:
  prometheus:
    image: prom/prometheus
    volumes:
      - ./monitoring/prometheus.yml:/etc/prometheus/prometheus.yml
    ports:
      - "9090:9090"
  grafana:
    image: grafana/grafana
    ports:
      - "3001:3000"
  node-exporter:
    image: prom/node-exporter
    ports:
      - "9100:9100"
```

### 2. 钉钉告警

`monitoring/dingtalk-alert.sh`：

```bash
#!/bin/bash
# 监控服务健康 + 钉钉告警
HEALTH=$(curl -s http://localhost:3000/api/health)
STATUS=$(echo $HEALTH | grep -o '"code":[0-9]*' | head -1 | cut -d: -f2)

if [ "$STATUS" != "0" ]; then
  # 钉钉 webhook 告警
  curl -X POST "https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN" \
    -H "Content-Type: application/json" \
    -d "{\"msgtype\":\"text\",\"text\":{\"content\":\"🚨 配送小智服务异常：$HEALTH\"}}"
fi
```

## 🆘 故障排查

### 查看日志
```bash
# 应用日志
docker compose logs -f app

# 数据库日志
docker compose logs -f db

# 进入容器调试
docker compose exec app sh
```

### 重启服务
```bash
docker compose restart app
```

### 数据库重置（**慎用，会丢数据**）
```bash
docker compose down -v   # 删除 volumes
docker compose up -d --build
```

### 备份恢复
```bash
# 1. 停止应用
docker compose stop app

# 2. 恢复
gunzip -c /var/backups/peisong/peisong_20260711_020000.sql.gz | \
  docker compose exec -T db psql -U peisong -d peisong

# 3. 重启
docker compose start app
```
