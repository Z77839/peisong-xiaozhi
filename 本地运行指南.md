# 配送小智 — 本地运行指南

> 🎯 **3 步启动，30 秒看到效果**

## 📋 前置条件

- **Node.js 18+**（推荐 20+）：https://nodejs.org/
- **Windows 10/11**（Mac/Linux 命令类似）

检查 Node 版本：
```powershell
node -v
# 看到 v18.x.x 或 v20.x.x 即可
```

## 🚀 启动（3 步）

### 步骤 1：解压代码

把代码解压到任意位置，例如：
```
C:\Users\35090\Desktop\peisong-xiaozhi\
```

### 步骤 2：双击 `start.bat`

在项目根目录双击 `start.bat`，它会：
- ✅ 自动检查 Node.js
- ✅ 第一次自动装依赖（1-3 分钟）
- ✅ 自动启动后端（端口 3000）
- ✅ 自动启动前端（端口 5173）
- ✅ 自动打开浏览器

### 步骤 3：登录

浏览器自动打开 `http://localhost:5173`

- 账号：`admin`（管理员）
- 密码：任意

3 个演示账号：
- `admin` — 系统管理员
- `operator` — 运营分析师
- `analyst` — 数据分析师

- 关闭那两个黑色窗口（后端/前端）
- 或双击 `停止.bat` 一键停

## 🔑 配置真实数据（可选）

1. 申请 Key：https://dev.qweather.com/
2. 在项目根目录创建 `.env` 文件（或编辑 `backend/.env`）：

## 🐛 常见问题

### Q1：双击 start.bat 后闪退？

A：用 PowerShell 跑看错误：
```powershell
cd C:\Users\35090\Desktop\peisong-xiaozhi
.\start.bat
```

### Q2：依赖装不上？

A：可能是网络问题。试试：
```powershell
cd backend
npm config set registry https://registry.npmmirror.com
npm install --legacy-peer-deps
```

### Q3：端口被占用？

A：3000 或 5173 被占时：
```powershell
# 找占用进程
netstat -aon | findstr :3000

# 杀掉
taskkill /F /PID <进程ID>
```

或修改端口：
- 后端：改 `backend/.env` 的 `PORT=3001`
- 前端：改 `frontend/vite.config.ts` 的 `server.port`

### Q4：登录失败？

A：后端没启动成功！检查后端窗口是不是有报错。

### Q5：看不到数据？

A：刷新一下（Ctrl+F5），等 1-2 秒加载。
